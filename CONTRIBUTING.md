# Contributing to Vibe Weekends

First off, thank you for considering contributing to Vibe Weekends! It's people like you that make Vibe Weekends such a great tool.

## Code of Conduct

This project and everyone participating in it is governed by the [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## How Can I Contribute?

### Reporting Bugs
Before creating bug reports, please check the existing issues to see if it has already been reported. When reporting a bug, use the Bug Report template and include as many details as possible (OS, exact steps to reproduce, console logs).

### Suggesting Enhancements
Enhancement suggestions are tracked as GitHub issues. When creating an enhancement request, use the Feature Request template, providing a clear and detailed explanation of the requested feature and why you think it would be useful.

### Pull Requests
The process described here has several goals:
1. Maintain Vibe Weekends's quality
2. Fix problems that are important to users
3. Engage the community in working toward the best possible product
4. Enable a sustainable system for Vibe Weekends's maintainers to review contributions

Please follow these steps to have your contribution considered by the maintainers:
1. Fork the repo and create your branch from `main`.
2. Follow all guidelines outlined in the PULL_REQUEST_TEMPLATE. 
3. If you've added code that should be tested, **add tests**.
4. Describe your changes clearly in the PR description, including how it was tested.
5. All PRs must be approved by at least one maintainer before merging.

### Local Development Setup
1. Fork the repo
2. Clone your fork
3. Run `npm install`
4. Run `npm run dev` to start the development server
5. Run `npm test` to verify your changes pass the test suite 

## Branch naming conventions
- `feat/feature-name`
- `fix/bug-name`
- `docs/documentation-update`
- `chore/chore-name`
